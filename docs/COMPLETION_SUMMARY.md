### 2️⃣ ETL DAG ПРИМЕРЫ (3 полных pipeline)

**etl_postgres_to_multi_db.py** (300+ строк кода)
```
Основной ETL pipeline:
- Извлечение данных из PostgreSQL
- Трансформация и валидация
- Параллельная загрузка в ClickHouse и Cassandra
- Проверка целостности данных
- Полное логирование всех операций

Расписание: Ежедневно в 02:00 UTC
```

**etl_csv_to_databases.py** (200+ строк кода)
```
CSV загрузка:
- Валидация структуры CSV
- Загрузка в ClickHouse
- Загрузка в Cassandra
- Обработка ошибок

Расписание: На спрос (Manual trigger)
```

**etl_hdfs_to_databases.py** (250+ строк кода)
```
Big Data интеграция:
- Чтение Parquet из HDFS через Apache Spark
- Трансформация в Spark
- Загрузка в оба хранилища
- Оптимизированная обработка больших объемов

Расписание: Еженедельно в пятницу
```

## ❓ FAQ

**Q: С чего начать?**
A: Откройте `START_HERE.md`, затем запустите `docker-compose up -d`

**Q: Как запустить DAG?**
A: Откройте http://localhost:8080, найдите DAG, нажмите play

**Q: Как добавить новый DAG?**
A: Создайте .py файл в папке `dags/` - Airflow автоматически подхватит

**Q: Как посмотреть логи?**
A: В Airflow WebUI или через `docker-compose logs -f`

**Q: Что делать при ошибке?**
A: Проверьте логи, посмотрите раздел "Решение проблем" в документации
